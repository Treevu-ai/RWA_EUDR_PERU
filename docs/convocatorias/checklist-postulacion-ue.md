# Checklist — postulación (cooperación UE / innovación temática)

Marca cada ítem cuando esté listo. Ajusta nombres de anexos al formato exigido por la convocatoria.

## Alineación y narrativa

- [ ] Problema claro: riesgo de mercado / cumplimiento / inclusión productiva (cifras o fuentes si el formulario lo permite).
- [ ] Objetivos **SMART** alineados con resultados esperados del programa (no solo entregables técnicos).
- [ ] Innovación explicada en **qué cambia** respecto a práctica actual (no solo “usamos blockchain”).
- [ ] Beneficiarios identificados (productores, cooperativas, exportadores) y **mecanismo de adopción** plausible.

## Consorcio y gobernanza

- [ ] Roles y responsabilidades por socio (si aplica).
- [ ] Capacidades demostrables del equipo (CV/resúmenes según pliegos).
- [ ] Plan de datos: titularidad, consentimiento, retención, agregación (RGPD / privacidad si la convocatoria lo exige).

## Factibilidad

- [ ] Hitos trimestrales con **entregables auditables**.
- [ ] Presupuesto coherente con actividades (personal, piloto campo, legal, difusión).
- [ ] Riesgos declarados + mitigaciones (adopción, integraciones, alcance normativo).

## Documentación del repositorio (base para redacción)

- [ ] Incorporar ideas de `docs/executive_summary.md` y `docs/overview.md` en la sección de resumen.
- [ ] Usar `docs/requirements.md` para el marco regulatorio (sin convertir en asesoría legal).
- [ ] Apoyar arquitectura y piloto con `docs/propuesta_integral.md` y `docs/roadmap.md`.

## Visibilidad UE (tras aprobación)

- [ ] Guía de visibilidad del programa descargada y revisada.
- [ ] Texto y emblema UE solo según **Grant Agreement** / anexo de comunicación (no anticipar en materiales oficiales antes de tiempo).

## Entrega final

- [ ] Límite de páginas / formato respetado.
- [ ] Firmas y anexos obligatorios completos.
- [ ] Copia de seguridad y versión archivada (fecha + hash git recomendable).
